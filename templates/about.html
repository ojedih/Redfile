About

Redfile is a user-friendly service to transfer any type of files between devices. That simple.

How it works?

1. The user selects a file and uploads it.

2. A code is generated to access the file from another device by scanning a QR or through a URL.

3. The user downloads the file.

How it works? But in depth.

After the user inputs a file and hits the "Upload" button, files go through an encryption algorythm which outputs encrypted bytes that are instantly uploaded to a Google Cloud Storage bucket.

A code is generated to access that file.

The user is then redirected to the page where they can either copy a URL or scan a QR to access the file from any device with a web browser installed.

Once the file is downloaded, it gets instantly deleted from Google Cloud Storage alongside with its database instance. No trace of information regarding that file is left behind.

What is the file size limit?

100 MB.

Nevertheless we're working on increasing that by implementing paid plans which will offer even more advantages.

Development

As mentioned above we want to expand by introducing paid plans so we can cover the hosting expenses. But there is more.

We're also planning on implementing an API for anyone who needs a service of storing temporary files on their project. Furthermore, since the service is open source we want to make a "kit" so anyone can install it on their local environment (home use, for example).
